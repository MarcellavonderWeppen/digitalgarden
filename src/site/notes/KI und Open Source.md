# KI und Open Source

Gestern Abend mal wieder meinen deutschen Lieblings-IT-YouTuber Morpheus geguckt:  

📺 [Die Open Source Community bricht zusammen](https://www.youtube.com/watch?v=yi1QiqlB1Q4) von _The Morpheus_

Was ich erfahren habe, hat mich sehr nachdenklich gestimmt: Die Open-Source-Bewegung ist kurz vorm oder mittem im Burn-Out!

KI zeigt auch hier ihre disruptive Wirkung und könnte die Spielregeln für immer ändern - und nicht zum Gutem. 

## Fundamentale Bedeutung von Open Source

Jahrezehntelang war Open Source das Fundament des Internets:

❗70 bis 90% beruhen auf offenem, frei zugänglichen Code - Projekte, die meistens von Entwicklern kostenlos in ihrer Freizeit gebaut und gepflegt wurden. 

Was vielen nicht bewusst ist: auch [[02 - Künstliche Intelligenz/2- Veröffentlicht/Open Source vs Proprietär\|proprietäre]], kommerzielle Software baut umfassend auf offenen Code auf - Arbeit, die aus reiner Liebe von der Open Source Community zur Verfügung gestellt wurde. 

Über 30 Jahre hat dies in der Welt der IT für einen sich selbstverstärkenden - unglaublich positiven - Kreislauf gesorgt: Programmierer haben ihren Code frei für alle zur Verfügung gestellt, so dass andere Programmierer diese Bausteine für eigene Projekte nutzen konnten. Open Source ist somit ein [[02 - Künstliche Intelligenz/2- Veröffentlicht/Common Good (Gemeingut)\|Common Good (Gemeingut)]] im Besten Sinne. 

## Bedrohung durch KI?

Nun droht dieses Prinzip zerstört zu werden. Schuld daran ist die KI - oder vielmehr der verantwortungslose Umgang damit. Denn weder Morpheus noch ich sind prinzipiell gegen KI. 

Eine der Ursachen der Krise ist das Aufkommen des sogenannten  [[Vibecoding\|Vibecoding]] KI kann ein wunderbares Tool in den Händen von erfahrenen Programmierern sein, die wissen was sie tun. Aber sie ermöglicht auch Dilettanten die schnelle Produktion riesiger Mengen von Code, der vordergründig betrachtet funktioniert, aber eigentlich fehleranfällig und schlecht durchdacht ist. Nun werden Pfleger von gemeinnützigen Projekten förmlich überschwemmt von “Beiträgen” solcher Art, die zwar in Sekunden erzeugt werden können, aber mühevoll von Menschen überprüft werden müssen.

Ein wesentliches Problem ist auch das ausbeuterische Verhalten profitorientierter Konzerne. Das Endziel scheint zu sein, den von Menschen geschriebenen Code ohne jede Gegenleistung zu nutzen für das Erschaffen einer Super-KI, die den Menschen überflüssig macht. Auf dem Weg dorthin maximalen Profit zu generieren.

## Lösungen in Sicht?

Ein interessanter Aspekt, den Morpheus ins Spiel bringt sind die Erkenntnisse von [[02 - Künstliche Intelligenz/2- Veröffentlicht/Elinor Ostrom und die Designprinzipien der Common Goods\|Elinor Ostrom]] zu den [[02 - Künstliche Intelligenz/2- Veröffentlicht/Common Good (Gemeingut)\|Common Goods]]. 

Sie hat intensiv erforscht, wie es kommt, dass manche Allemende (Gemeingüter) übernutzt und dadurch zerstört werden, wohingegen andere über Jahrhunderte nachhaltig zum Wohlstand der Gemeinschaft beitragen. Dabei haben sich acht [[02 - Künstliche Intelligenz/2- Veröffentlicht/Elinor Ostrom und die Designprinzipien der Common Goods\|Designprinzipien für Gemeingüter]] herauskristallisiert. 

Morpheus erwähnt diese Prinzipien kurz. Mein Gefühl sagt: Hier könnte der Schlüssel zum Problem sagen und es würde sich lohnen, genau dort weiter zu denken. 

Es ist wirklich eine ungeheuerliche Schieflage, wenn milliardenschwere Konzerne ohne jeden Dank, Respekt oder gar eine  Gegenleistung dieses Common Good ausbeuten - mit dem Ziel der sie sich einzuverleiben und in diesem Prozess zu zerstören. 

Da macht schon das erste Prinzip von [[02 - Künstliche Intelligenz/2- Veröffentlicht/Elinor Ostrom und die Designprinzipien der Common Goods\|Ostrom]] Resonanz: 

> Klar definierte Grenzen: Die Grenzen der Ressource und der Personen, die Nutzungsrechte haben, müssen klar definiert sein.

Es ist einfach nicht fair, wenn kommerzielle Projekte sich genauso wie altruistische motivierte Programmierer sich kostenlos an diesem frei zugänglichen Code bedienen.
